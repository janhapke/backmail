## Implementation Notes

### ESM Interop Patterns

Both imapflow and simple-git use different ESM export strategies:

1. **imapflow (CommonJS, no exports field)**
   - Imported: `import { ImapFlow } from 'imapflow'`
   - Node.js ESM allows this via implicit ESM-to-CJS adapter
   - Works cleanly under `moduleResolution: nodenext`

2. **simple-git (Dual ESM+CJS)**
   - Imported: `import { simpleGit } from 'simple-git'` (named export)
   - Prior pattern (v2): default import caused TypeScript errors in nodenext
   - v3 ships dedicated `dist/esm/index.js` with named exports — this is correct

| Decision | Rationale | Impact |

| Exact version pins (no `^` ranges) | Prevent npm-time supply-chain drift in CI/downstream | Requires explicit `npm install @version` for future bumps |
