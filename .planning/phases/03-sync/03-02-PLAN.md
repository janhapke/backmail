    **syncAccount(accountName, config, opts): Promise<SyncResult>** (SYNC-01/02/03/04/05)
    1. If opts.onlyFolders.length>0 && opts.excludeFolders.length>0: throw `Error('--only-folder and --exclude-folder are mutually exclusive')`
    2. password = await getPassword(accountName)
    3. repoInitialized = await ensureRepo(config.repoPath)
    4. await fs.mkdir(path.join(config.repoPath, 'messages'), { recursive: true })
    5. await fs.mkdir(path.join(config.repoPath, 'folders'), { recursive: true })
    6. client = new ImapFlow({ host, port, secure: tls, auth:{user:username, pass:password}, logger: false })
    7. Accumulators: added=0, removed=0, partial=false, folderResults: FolderSyncResult[] = []
    8. try:
       - await client.connect()
       - raw = await client.list()
       - folders = filterFolders(raw, opts.onlyFolders, opts.excludeFolders)
       - for each folder: call syncFolder(client, folder, config.repoPath, opts.verbose)
         - read `folders/<folderPathToFilename(folder.path)>.json` if exists then parse to storedState: FolderState|null
         - wrap in getMailboxLock; inside: uidvalidity compare (BigInt) — if mismatch: delete every local .eml whose message-id is in storedState.messages, set storedState=null (full re-sync)
         - lastUid = storedState && storedState.messages.length>0 ? Math.max(...uids) : 0 (Pitfall 2)
         - range = lastUid===0 ? '1:*' : `${lastUid+1}:*` (Pitfall 3)
         - for await (msg of client.fetch(range, {uid:true, source:true, envelope:true, flags:true}, {uid:true})):
           - rawId = msg.envelope?.messageId ?? `no-message-id_uid-${msg.uid}_${folderPathToFilename(folder.path)}`
           - safeId = sanitizeMessageId(rawId)
           - fs.writeFile(path.join(repoPath,'messages',`${safeId}.eml`), msg.source)
           - push to newMessages: { uid: msg.uid, 'message-id': rawId, flags: Array.from(msg.flags) }
         - serverUids = new Set(await client.search({all:true},{uid:true}))
         - existing = storedState?.messages ?? []
         - kept = existing.filter(m => serverUids.has(m.uid))
         - removedList = existing.filter(m => !serverUids.has(m.uid))
         - for each m in removedList: fs.unlink(path.join(repoPath,'messages',`${sanitizeMessageId(m['message-id'])}.eml`)).catch(()=>{})
         - updatedState: FolderState = { uidvalidity: client.mailbox.uidValidity.toString(), uidnext: client.mailbox.uidNext, messages: [...kept, ...newMessages] }
         - fs.writeFile(folders/<safeName>.json, JSON.stringify(updatedState, null, 2))
         - lock.release() in finally
         - return { path: folder.path, added: newMessages.length, removed: removedList.length }
         - On per-folder error: push { path, added:0, removed:0, error: err as Error } to folderResults and continue — do NOT abort the whole account unless the error is a connection-level error
       - Accumulate: added += folderResult.added, removed += folderResult.removed
    9. catch (err): if (added>0 || removed>0) partial=true else throw err (D-09)

    11. git = simpleGit(config.repoPath); status = await git.status(); if (!status.isClean()) { await git.add('.'); await git.commit(formatCommitMessage(added, removed, partial)) } (Pitfall 5 — skip commit when clean)
    12. return { added, removed, partial, repoInitialized, folderResults }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IMAP server → local filesystem | IMAP-supplied Message-ID and folder path are used as filesystem path components. Untrusted input crosses here. |
| IMAP server → process memory | Password supplied to ImapFlow; imapflow's default logger could echo auth exchanges. |
| filesystem (messages/, folders/) → git working tree | All written files are committed via simple-git `add('.')`. Any path traversal succeeds into git history too. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Tampering | `sanitizeMessageId()` consumers (`fs.writeFile`, `fs.unlink` under messages/) | mitigate | `sanitizeMessageId()` strips angle brackets, replaces every char in `/\\:*?"<>|` with `_`, replaces `..` with `__`, caps length at 200. Verified by unit tests A-E in tests/unit/sync.test.ts. |
| T-3-02 | Tampering | `folderPathToFilename()` consumers (`fs.writeFile` under folders/) | mitigate | `folderPathToFilename()` replaces every char in `/\\:*?"<>|` with `_`, replaces `..` with `__`. Verified by unit tests F-H. |
| T-3-03 | Information Disclosure | ImapFlow default pino logger | mitigate | `logger: false` MANDATORY in every ImapFlow constructor call. Acceptance criteria grep-enforces the literal string. Password never logged by core (ARCH-01 forbids console.*). |
| T-3-04 | Denial of Service | Unbounded `.eml` body write | accept | Low-risk for v1 single-user CLI. Message size bounded by IMAP server limits. Revisit if multi-gigabyte mail observed in practice. |
| T-3-05 | Repudiation | Commit identity | accept | Handled by user's git identity config; open question documented in RESEARCH.md Open Q #2 — error path surfaces a helpful message. Not a v1 security requirement. |
| T-3-06 | Elevation of Privilege | simple-git shelling out to git binary | mitigate | simple-git handles argument quoting; we never concatenate user input into git commands (we only pass repoPath to `simpleGit()` and a formatted commit message string). ASVS L1 V5.3 satisfied. |
</threat_model>
