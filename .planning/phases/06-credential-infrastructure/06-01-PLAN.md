<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → loadRepositoryConfig | .backmail/config.json is a user-controlled file read from disk |
| process.env → getPasswordByRef | Environment variables are caller-controlled at process spawn time |
| OS keyring → getPasswordByRef | Keyring query result is OS-controlled; may return null or throw |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | loadRepositoryConfig / config.json | mitigate | Zod schema validates all fields — invalid types or missing required fields throw ZodError before any credential lookup occurs |
| T-06-02 | Information Disclosure | getPasswordByRef / keyring | accept | Credential is read from OS keyring; no logging of resolved password (ARCH-01 forbids console.* in core); caller (CLI) decides how to surface errors |
| T-06-03 | Information Disclosure | getPasswordByRef / env var | accept | BACKMAIL_PASSWORD is a deliberate, documented opt-in for CI; no mitigation needed beyond clear documentation |
| T-06-04 | Tampering | parsePasswordRef / URLSearchParams | mitigate | Semicolon-to-ampersand replacement is performed before URLSearchParams parsing; strict check for missing service= and account= keys prevents silent empty-string lookups |
| T-06-05 | Denial of Service | loadRepositoryConfig / ENOENT | accept | Missing config file produces a friendly error with actionable message; no infinite retry, no process.exit in core |
| T-06-06 | Elevation of Privilege | getPasswordByRef / unknown scheme | mitigate | Unrecognised scheme throws immediately per D-04 — no partial credential lookup attempted on unknown input |
</threat_model>
