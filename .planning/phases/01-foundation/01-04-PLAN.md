```bash
#!/usr/bin/env bash
# scripts/test-integration.sh
# Orchestrates Docker-backed integration tests (D-04).
# Usage: bash scripts/test-integration.sh (invoked by npm run test:integration)
set -euo pipefail

cleanup() {
  echo "Tearing down Docker..."
  docker compose down --remove-orphans
}
trap cleanup EXIT

echo "Starting IMAP container..."
docker compose up -d

echo "Waiting for IMAP to be ready on port 143..."
timeout 60 bash -c 'until docker compose exec -T imap nc -z localhost 143 2>/dev/null; do sleep 1; done'

echo "Running integration tests..."
npx vitest run --config vitest.integration.config.ts

echo "Done."
```

Key constraints to verify in the script:

- Readiness probe uses `nc -z localhost 143` inside the container, NOT a fixed `sleep N`
- `timeout 60` prevents infinite hang if the container fails to start
- `docker compose down --remove-orphans` removes any orphaned containers from previous interrupted runs
- Uses `npx vitest run --config vitest.integration.config.ts` — not `npm test` (which uses the unit config)

Create the directory `tests/integration/` and write `tests/integration/imap-connect.test.ts` (per TEST-01, RESEARCH.md Validation Architecture):

```typescript
import { describe, it, expect } from 'vitest'
import * as net from 'node:net'

// Allow override via env vars for CI environments that map ports differently
const IMAP_HOST = process.env.IMAP_HOST ?? 'localhost'
const IMAP_PORT = Number(process.env.IMAP_PORT ?? '143')

describe('TEST-01: IMAP container connectivity', () => {
  it('TCP connection to IMAP port succeeds', () => {
    return new Promise<void>((resolve, reject) => {
      const socket = net.connect(IMAP_PORT, IMAP_HOST, () => {
        socket.destroy()
        resolve()
      })
      socket.on('error', reject)
    })
  })

  it('IMAP greeting contains Dovecot ready banner', () => {
    return new Promise<void>((resolve, reject) => {
      const socket = net.connect(IMAP_PORT, IMAP_HOST, () => {
        socket.once('data', (data) => {
          const banner = data.toString()
          // RFC 3501: server greeting starts with "* OK"
          expect(banner).toMatch(/\* OK/)
          socket.destroy()
          resolve()
        })
      })
      socket.on('error', reject)
    })
  })
})
```

Key constraints:
- Uses raw `node:net` TCP — no imapflow dependency in Phase 1 (imapflow is added in Phase 2)
- `IMAP_HOST`/`IMAP_PORT` env vars allow CI to override without code change
- Default values match docker-compose.yml port mapping: localhost:143
- Checks `* OK` RFC 3501 IMAP greeting prefix — the minimal-imap/Dovecot container sends this on connect
- Both tests return Promises (Vitest handles async tests that return Promise)
- Only run via `npm run test:integration` (which uses vitest.integration.config.ts) — NOT via `npm test`

| Threat ID | Category | Component | Disposition | Mitigation Plan |
