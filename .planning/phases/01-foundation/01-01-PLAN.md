Note on D-01: CONTEXT.md mentions `"bundler"` as an option alongside `"node16"`. The correct choice for a Node.js CLI distributed via npm is `nodenext` — it enforces `.js` extensions on relative imports, preventing ERR_MODULE_NOT_FOUND when running `dist/` output directly. `bundler` is only appropriate for Vite/webpack pipelines.

```typescript
    testTimeout: 120_000,
    hookTimeout: 30_000,
  },
})
```

Key constraints:
- vitest.config.ts MUST NOT include `tests/integration/**` — integration tests have Docker dependency and must not run in `npm test`
- vitest.integration.config.ts testTimeout=120_000 is required because Docker cold pull can take 30-60s on CI (RESEARCH.md Pitfall 2)
- Both files use `defineConfig` from `vitest/config`, not from `vite`
