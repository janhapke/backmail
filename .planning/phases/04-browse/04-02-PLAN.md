    Implement the following functions in src/core/browse.ts:

    1. **getLog(repoPath: string, limit: number | 'unlimited'): Promise<string[]>**
       - Call simpleGit(repoPath).log()
       - Pass maxCount: limit if limit is number, omit maxCount if limit === 'unlimited'
       - Filter results to sync commits only using pattern: /^\d{4}-\d{2}-\d{2}(\s\[partial\])?:\s+\+\d+\s+added\s+\/\s+-\d+\s+removed$/
       - Map results to c.message (the commit message IS the display string per D-04)
       - Return as Promise<string[]>
       - Per 04-RESEARCH.md section 3, sync commits have format: "YYYY-MM-DD: +N added / -N removed" or "YYYY-MM-DD [partial]: +N added / -N removed"

    2. **Helper: isDateString(arg: string): boolean**
       - Return /^\d{4}-\d{2}-\d{2}$/.test(arg)
       - Used by checkoutCommit to detect date vs. commit hash

    3. **Helper: resolveDate(git: SimpleGit, dateStr: string): Promise<string>**
       - Query git log with:
         - '--after': `${dateStr} 00:00:00`
         - '--before': `${dateStr} 23:59:59`
         - '--max-count': '1' (newest first, so first result = last commit of the day)
       - If log.total === 0, throw new Error(`No sync commit found for date ${dateStr}`)
       - Return log.latest!.hash
       - Reference: 04-RESEARCH.md section 4

    4. **checkoutCommit(repoPath: string, dateOrHash: string): Promise<{path: string, sha: string}>**
       - Detect if dateOrHash is a date string using isDateString()
       - If date: resolve to commit hash using resolveDate()
       - If commit hash: use as-is
       - Determine worktree name (D-10):
         - Date input → worktree name is the date string (e.g., "2026-04-22")
         - Commit hash → worktree name is first 7 chars of hash (e.g., "abc1234")
       - Construct worktree path: path.join(repoPath, '.worktrees', worktreeName)
       - Overwrite existing worktree (D-07):
         - Try to remove existing: await git.raw(['worktree', 'remove', '--force', worktreePath]).catch(() => {})
         - Force-remove directory: await fs.rm(worktreePath, { recursive: true, force: true }).catch(() => {})
       - Create new worktree: await git.raw(['worktree', 'add', worktreePath, commitHash])
       - Ensure .gitignore entry (D-06): call ensureWorktreesIgnored(repoPath)
       - Return { path: path.resolve(worktreePath), sha: commitHash.slice(0, 7) }
       - Output will be formatted in CLI (per D-09): "Checked out <date-or-hash> (<short-sha>) → <absolute-path>"

    5. **Helper: ensureWorktreesIgnored(repoPath: string): Promise<void>**
       - Read .gitignore at path.join(repoPath, '.gitignore'), default to empty string if not found
       - If content does not include '.worktrees/', append it:
         - If content ends with '\n', append '.worktrees/\n'
         - Otherwise, append '\n.worktrees/\n'
       - Write back to .gitignore using fs.appendFile or fs.writeFile (prefer appendFile per 04-RESEARCH.md)

    Implementation notes:
    - Use simpleGit per 04-PATTERNS.md section "simple-git Usage"
    - For worktree operations, use git.raw(['worktree', ...]) — no typed API in simple-git 3.x
    - Follow error handling pattern from sync.ts: throw descriptive errors, let CLI catch them
    - All paths are absolute (use path.resolve when needed)
    - All async operations should be awaited

            ```typescript
            const parts: Array<{type: string; content: string}> = []

            for (const att of parsed.attachments ?? []) {
              parts.push({type: att.contentType, content: att.content.toString('base64')})
            }
            ```
          - Return as: { headers, parts } (Record<string, unknown>)
          - Per spec, this is the JSON format mapping from 04-RESEARCH.md section 1
