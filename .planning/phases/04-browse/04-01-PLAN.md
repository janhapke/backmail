<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → dateOrHash arg | Date string and commit hash arguments in checkout command will be passed through, potential for malformed refs |
| file system access → messages/ | browse.ts will read .eml files, potential for path traversal if messageId not sanitized |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Tampering | uidvalidity JSON handling | mitigate | Removed unsafe BigInt() conversion; now uses string comparison (CR-01 fix in sync.ts) — eliminates parse attack surface |
| T-4-02 | Information Disclosure | browse functions read arbitrary .eml files | mitigate | All browse functions receive repoPath and construct paths safely; apply sanitizeMessageId(messageId) before filesystem lookup (T-3-01 reused from sync.ts) |
| T-4-03 | Elevation of Privilege | git worktree creation with arbitrary ref | mitigate | Worktree path is constructed by checkout function using `.worktrees/<safe-name>` pattern; simple-git.raw(['worktree', 'add', path, ref]) validates ref internally — will error on malformed refs |
</threat_model>
