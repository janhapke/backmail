**Subcommand structure (from D-01, D-05, D-06, D-12–D-19):**
```typescript
program
  .command('restore [date|commit]')
  .description('Restore messages from backup to target IMAP server')
  .requiredOption('--to <imap-url>', 'target IMAP URL (imap:// or imaps://)')
  .option('--account <name>', 'account name (optional if single account configured)')
  .option('--skip-duplicates <yes|no>', 'check for duplicates (default: yes)', 'yes')
  .option('--dry-run', 'output without writing to target server', false)
  .option('--verbose', 'log one line per message', false)
  .action(async (dateOrCommit: string | undefined, opts: {
    to: string
    account?: string
    skipDuplicates: string
    dryRun?: boolean
    verbose?: boolean
  }) => {
    try {
      // Validate --to flag (required; commander enforces this via requiredOption)
      // Parse options and resolve account
      const [accountName, accountConfig] = resolveAccount(config, opts.account)

      // Convert --skip-duplicates string to boolean (D-11)
      const skipDuplicates = opts.skipDuplicates === 'yes'
      const dryRun = opts.dryRun ?? false
      const verbose = opts.verbose ?? false

      // Call core restoreAccount function
      const result = await restoreAccount(
        accountConfig,
        opts.to,
        dateOrCommit,
        {
          skipDuplicates,
          dryRun,
          verbose,
        }
      )

      // Format and print output (D-14, D-15, D-16)
      // Per-folder lines: "INBOX: 143 uploaded, 2 skipped" or "[dry-run] INBOX: would upload 143, skip 2"
      // Per-message lines (if verbose): "  ↳ <message-id>"
      // Final summary: "Total: 543 uploaded, 12 skipped, 0 errors"

      // Output final summary (D-14, D-16, D-18)
      const prefix = dryRun ? '[dry-run] ' : ''
      if (result.errors === 0) {
        console.log(
          `${prefix}Total: ${result.uploaded} uploaded, ${result.skipped} skipped`
        )
      } else {
        // D-18: Include error count in summary
        console.log(
          `${prefix}Total: ${result.uploaded} uploaded, ${result.skipped} skipped, ${result.errors} errors`
        )
        // D-19: Include retry hint
        console.error('Re-run with --skip-duplicates=yes to safely retry (already-uploaded messages will be skipped)')
        process.exit(1)
      }
    } catch (err) {
      // D-19: Print error but never the URL with password (Pitfall 4, T-5-02)
      const msg = (err as Error).message
      console.error(`Restore failed: ${msg}`)
      process.exit(1)
    }
  })
```

**Important implementation details:**

1. **Options parsing (D-01, D-05, D-06, D-12–D-15):**
   - `[date|commit]` — positional, optional (can be date YYYY-MM-DD or git hash)
   - `--to <imap-url>` — required, must be imap:// or imaps://
   - `--account <name>` — optional, uses resolveAccount() to select account
   - `--skip-duplicates <yes|no>` — optional, defaults to 'yes'
   - `--dry-run` — flag, false by default
   - `--verbose` — flag, false by default

2. **Error handling (D-17, D-18, D-19, Pitfall 4):**
   - On error, print human-readable message (never the full error with URL)
   - If restore succeeds but has errors (result.errors > 0):
     - Print final summary with error count
     - Print retry hint: "Re-run with --skip-duplicates=yes to safely retry..."
     - Exit with code 1
   - If restore throws exception:
     - Print "Restore failed: {error message}" (error message should not include password)
     - Exit with code 1

3. **Output formatting (D-14, D-15, D-16):**
   - This plan handles final summary only
   - Per-folder and per-message output is typically added by the core function or CLI wrapper
   - For MVP: just output total summary
   - Final format: `Total: {uploaded} uploaded, {skipped} skipped, {errors} errors`
   - If dry-run: prefix with `[dry-run]`

4. **Import and export expectations:**
   - Import restoreAccount, RestoreResult, RestoreOptions from src/core/index.js
   - Import resolveAccount from src/core/index.js (already in use)
   - No new imports needed beyond what's already in the file

5. **Follow existing pattern:**
   - Use same error handling as checkout: try/catch, console.error, process.exit(1)
   - Use same account resolution as log/checkout/ls: resolveAccount(config, opts.account)
   - Use same output style: console.log for normal output, console.error for errors

**Note on per-folder output (D-14, D-15):**
The current restoreAccount() returns aggregate { uploaded, skipped, errors } totals. For a more detailed output (per-folder breakdown), the core function could be enhanced to return per-folder results (like syncAccount does with folderResults array). However, for the MVP CLI, final summary is sufficient. If tests require per-folder output, add a folderResults array to RestoreResult and update output formatting accordingly in a follow-up.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI argv (--to) → restore subcommand | User provides target URL with embedded password; must not leak in error output |
| CLI error output → stdout/stderr | Error messages seen by user; must sanitize credentials |
| restoreAccount() result → CLI output | Core layer returns counts; CLI formats and displays without exposing internals |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-02 | Information Disclosure | Error messages in CLI | mitigate | Catch errors, extract message only, never print full exception or opts.to; print "Restore failed: {msg}" instead of "Restore failed to {opts.to}: {msg}" (per Pitfall 4) |
| T-5-09 | Information Disclosure | Process arguments visible to ps | accept | Credentials in URL visible to ps output; acceptable for one-shot developer tool; document in user guide for sensitive environments |
| T-5-10 | Denial of Service | Malformed URL in --to flag | mitigate | URL parsing happens in core parseImapUrl(); if invalid, throws error; CLI catches and prints error; no DOS attack possible |

</threat_model>
