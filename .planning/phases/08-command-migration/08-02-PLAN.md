<objective>
Remove all account-registry remnants from the CLI entry point: the `getConfig()` helper, the `accounts` subcommand, the `resolveAccount` dead import, the `--account <name>` option from every command, and `--all` from the sync command.

Purpose: The CLI currently registers options and commands that reference the abolished multi-account model. Commander will produce "unknown option" / "unknown command" errors for these if a user passes them, which is fine per D-01/D-02/D-03 — but the option/command registrations themselves must be gone so they do not appear in `--help` output and do not confuse future readers.

Output: `src/cli/index.ts` that registers only options that actually do something: `--workdir`, `--exclude-folder`, `--only-folder`, `--verbose`, `--limit`, `--format`, `--to`, `--skip-duplicates`, `--dry-run`.
</objective>
